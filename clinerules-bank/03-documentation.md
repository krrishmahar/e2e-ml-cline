# Documentation Rules

## 1. Structure
- Every function must have docstrings.
- Use Markdown tables for parameter descriptions.

## 2. Auto-doc Integration
- Coderabbit will generate:
  - docs/api/
  - docs/architecture/
  - CHANGELOG.md
- Trigger @gh_docs for coderabbit
- Place all docs in docs/

## 3. PR Template
- Summary
- Changes
- Expected Output
- Risks
